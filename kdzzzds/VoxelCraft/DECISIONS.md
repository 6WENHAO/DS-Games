# VoxelCraft 关键决策记录

## Phase 1 — 内外无缝场景骨架
- **房间结构**：体素块建房间，配置数据驱动（room.js ROOM_CONFIG）
- **外墙**：白玉块（quartz 61），现代感
- **地板**：樱花木板（31）+ 红色羊毛地毯（78）
- **南墙**：全幅落地玻璃（63），无框，Phase 2 做镜面
- **北墙**：开放出入口 3×3，无缝连通外部世界，无加载/雾门/传送
- **家具**：壁炉、中央桌、西墙书架×2、东墙沙发+工作台+箱子+熔炉、窗边盆栽×2
- **天花板**：白玉块 + 4 盏萤光石灯（48）
- **性能**：World.stampBlocks() 批量写入→一次性 remesh（原逐块setBlock会卡死）
- **建筑位置**：spawn + offset，自动探测地形高度；门外 3 格清空铺通道
- **玩家出生**：室外北侧，面朝南（yaw=π），步入房间后面对镜面墙

## Phase 2 — 镜像系统
- **方案**：平面反射相机 + RenderTarget（Reflector 模式），直接落地
- **反射面**：单一大 Quad（15×6 世界单位），覆盖南墙全部玻璃块
- **反射相机**：主相机位置/朝向关于镜面平面反射
- **着色器**：投影纹理 UV（projView 矩阵），避免 UV 映射错位
- **图层隔离**：镜面渲染时隐藏自身 mesh 避免自反射
- **分辨率**：512×224，后续可根据 FPS 调低
- **主相机**：排除 CHAR_LAYER（layer 1），使角色 mesh 在第一人称不显示
- **反射相机**：默认渲染全部层（含 CHAR_LAYER），角色/NPC 全部进镜像

## Phase 3 — 三角色
- **切换**：C 键循环 人类↔大猩猩↔小鸡
- **碰撞差异化**：
  - 人类：0.3半宽 × 1.8高，眼高 1.62
  - 大猩猩：0.45半宽 × 1.6高，眼高 1.4（宽矮）
  - 小鸡：0.15半宽 × 0.65高，眼高 0.5（小体型）
- **模型**：Three.js 基元组装（Box/Sphere/Cone），无骨骼
- **动画**：sin 波驱动四肢摆动，走路/空闲自动切换
  - 人类：臂腿交替摆动
  - 猩猩：长臂大幅前后摆 + 身体左右摇摆
  - 小鸡：腿快速交替 + 身体弹跳 + 周期性啄食
- **图层**：角色 mesh 在 CHAR_LAYER，主相机排除该层，反射相机包含

## Phase 4 — NPC 批量同步
- **召唤**：N 键每次召唤 3 个 NPC（随机角色类型）
- **巡逻**：随机方向行走，定时转向（1.5~4.5s），遇墙自动折返
- **碰撞**：与玩家相同逻辑（分轴检测），适配各角色体型
- **重力**：简单重力下落，站在实心方块上
- **动画**：复用 Characters.animate，根据移动状态切换
- **镜像**：NPC mesh 同处 CHAR_LAYER，反射相机自动捕获

## 优化轮（借鉴 Minecraft 模组 / 开源 voxel 引擎源码）
研读对象：**Sodium**（CaffeineMC，渲染优化模组）、**Lithium**（CaffeineMC，逻辑优化模组）、**noa-engine**（fenomas，JS voxel 引擎）

### mesher.js（借鉴 Sodium CompactChunkVertex + noa terrainMesher/0fps AO）
- **逐角 AO 环境光遮蔽**：noa `packAOMask` 算法（2bit×4 角打包进贪婪 mask，合并要求 id+AO 全等）；
  对角线按 AO 翻转（`a0+a2 > a1+a3` 时沿 v1-v3 切分），消除各向异性噪点；AO 亮度预乘进 shade 顶点色（Sodium 做法）
- **AO 距离淡出**：额外 shade2（无 AO 亮度）属性，顶点着色器 24→56m smoothstep 混合
  —— 近处有体积感/接触阴影，>60m 与原版逐像素一致（A/B 截图对比验证，远景带 diff 从 9-11% 降至 1-4%，其余为 56m 内树冠）
- **压缩顶点格式**：位置 Int16 定点（×16，着色器 ×0.0625 还原）、uv Int16、tile Uint8、shade Uint8 归一化
  —— 每顶点 32B → 13B；索引 vc≤65535 时用 Uint16；实测传输/显存体积 **-39%**（虽 AO 拆分使 quad +46%）
- **体素直接寻址**：Worker 直接复用 genChunk 的 pad 缓冲，主线程编辑时一次快照，热循环零闭包调用
  —— 49 区块网格构建 87ms → 44ms（**约 2 倍**）
- **Worker 侧预计算 y 包围范围** → 主线程直接构造 boundingSphere，避免 three 惰性扫描顶点引起首帧卡顿（Sodium：一切离渲染线程）

### world.js / lod.js
- 任务队列**排序节流**：仅在入队/过滤后或 400ms 超时重排（原每次 pump 全量 sort）（Lithium：去热路径冗余）
- LOD 瓦片同样返回 y 范围 → 预计算 boundingSphere，视锥剔除零首帧成本
- 远景瓦片 clip 材质切换加**状态缓存**，避免每帧重挂材质

### mirror.js（借鉴 three.js Reflector / Immersive Portals 传送门剔除）
- **可见性剔除**：镜面不在视锥 / 相机在背面 / 距离 >110m → 跳过整帧反射渲染
- **帧率节流**：28m 内全帧率，之外隔帧（纹理保留上一帧）
- **斜切近平面**（oblique frustum）：镜面平面作为反射相机近裁剪面，反射不再透出镜后几何且省填充率
- **远平面钳制** 2600m：反射相机视锥剔除大量远景 LOD
- 临时向量全部复用，零每帧分配

### npc.js / player.js（借鉴 Lithium 实体/碰撞）
- 碰撞热循环 `World.getBlock` 每格只调一次（原重复两次）；角色配置建 NPC 时缓存
- **距离分级更新**：>48m 隔帧、>96m 每 4 帧（积累 dt 步进，速度不变）；>64m 跳过动画
- 玩家 raycast 临时 Vector3/Euler 提升为模块级复用

### 验证
- Node 单测：新旧两路径（Worker pad / 主线程 gb）输出完全一致；索引、定点范围、AO 多级、y 包围全部通过
- `npm run shot` 无头截图 3 张全过、0 JS 错误；像素级检查镜面反射内容、地面 AO 层次存在
- 运行时自检：方块放置/破坏/跨区块编辑/爆炸、NPC 召唤巡逻、镜面正背面切换，全部通过

### 远景观感修复（用户反馈轮）
- **巨树忽大忽小**：原 L3+ 层把巨树放大成"远景地标"（+16 高、冠幅最大 60+ 格），环切换时体积跳变。
  改为全层级统一真实冠幅（gwide=9、顶高 h+th+3），Node 自检 s=2..32 顶高完全一致
- **LOD 环方形分界线**：三项成因逐一消除——
  ① 逐层阶梯下沉（0.9/1.8/3.6/7.2m）→ 顶点着色器按距离连续下沉（0.45·cheb/256，各层边界处沉降相同，无台阶）
  ② 边界硬切割 → 互补点抖交叉淡化（stipple crossfade）：细层外缘 `aOut≥dn` 与粗层内缘 `aIn≥1-dn` 共用同一屏幕噪声、
    过渡带对齐同一区间（bandO_l = bandI_{l+1} = 24·2^l），每像素恰好一层，无缝且无双重混合
  ③ 水面层级台阶/重叠叠加变暗 → 各层水面完全共面 + depthWrite 开启（等深度测试保证单层混合），LOD 水 alpha 统一 0.72 与近景一致
  实测（800m 俯视纯海面径向亮度剖面）：512m 边界亮度跳变 17.3 → 8.0，原"暗色峡谷环"（-20 亮度带）消除

## 限高 3000 + 高原群系（写实还原轮）
### 优化算法（先规划后实现）
- **变高度区块存储**：区块 blocks/pad 步幅改为按需 `H = 本区块最高地形 + 44`（平原 ~25KB，峰区 ~550KB，
  对比固定 3000 高的 768KB —— 平原内存降 30 倍）；越高编辑触发 `growChunk` 逐列扩容
- **hmin 跳层网格**：genChunk 返回 pad 窗口最低地表，网格器三轴扫掠从 `hmin-1` 起跳（其下全实心零面），
  峰区区块 47→37ms、高原面 23→9ms、草甸 14→3ms；跨界挖掘经 `lowerHmin` 同步相邻区块
- **高海拔岩体快速路径**：y>150 且距地表 >8 的内部直接置岩（免逐块矿物哈希），阶地群系(h<150)不受影响
- **顶点定点改 Uint16**：×16 定点最高支持 y=4095，实测峰顶 y≈2142 → 34272 在量程内
- 索引 vc>65535 自动 Uint32（巨型峭壁区块）；LOD/包围球/雾均自适应新高度

## 积雪位置优化轮（以显示为准）
### 规则（近景 genChunk / 远景 buildLodTile 同源，gen.js 单一来源）

- **雪地针叶林（修"树下漏雪"）**：B_SNOW 雪区允许杉树（密度 0.008，非雪区 0.014 不变），
